# Functional Requirements Document (FRD)
## موقع كلية الآثار — جامعة القاهرة (إعادة البناء)

**الإصدار:** 1.0  
**التاريخ:** يونيو 2026  
**الحالة:** مسودة أولى

---

## 1. نظرة عامة على النظام

الموقع الجديد هو منظومة متكاملة تخدم كلية الآثار — جامعة القاهرة. يتكون النظام من **طبقتين رئيسيتين متصلتين**:

- **Frontend (واجهة المستخدم):** موقع عام بنسختين عربي/إنجليزي
- **Backend (لوحة إدارة):** CMS متكامل يتحكم في كل محتوى الموقع

كل وحدة وظيفية في النظام **مرتبطة بـ CMS مركزي واحد** — أي تعديل من الإدارة ينعكس فوراً على الموقع العام.

## 1.1 البوابات الثلاث والـ Dashboard

النظام يتكون من **ثلاث بوابات مستقلة** لكل منها Navbar وDashboard خاص بها بناءً على نوع المستخدم. نقطة الدخول الموحدة هي صفحة تسجيل الدخول التي تُوجّه كل مستخدم للبوابة المناسبة.

```
/login  →  يكشف role من الـ JWT
            ├── student       → /student/dashboard
            ├── faculty       → /faculty/dashboard
            └── admin         → /admin/dashboard
```

## 1.2 كروت الأقسام الأربعة (Department Cards)

**الموقع في الصفحة الرئيسية:** قسم مستقل بعد الـ Hero Section مباشرة.

**الوصف:** أربعة كروت تمثل الأقسام العلمية الأربعة للكلية، مصممة بـ Hover Animation تعطي انطباعاً بصرياً عميقاً وتعكس هوية كل قسم.

### تفاصيل كل كارت

| القسم | الـ Slug | اللون المميز | الأيقونة المقترحة |
|-------|---------|------------|-----------------|
| الآثار المصرية | `egyptology` | ذهبي (#C9A84C) | هيروغليف / أهرامات |
| الآثار الإسلامية | `islamic` | أخضر زيتي (#4A7C59) | زخرفة إسلامية / قبة |
| ترميم الآثار | `conservation` | بني تراب (#8B6914) | فرشاة ترميم / متحف |
| الآثار اليونانية الرومانية | `greco-roman` | أزرق بحري (#2C5282) | عمود إغريقي / تمثال |

### سلوك الـ Hover Animation

```
الحالة الافتراضية (Default State):
  - الكارت يعرض: اسم القسم + أيقونة + لون خلفية هادئ
  - ارتفاع الكارت: ثابت
  - الصورة: مخفية أو بشفافية 30%

عند الـ Hover:
  - transform: translateY(-8px)          -- ارتفاع الكارت للأعلى
  - box-shadow: تظليل عميق
  - الصورة الخلفية: opacity من 30% → 80%  -- ظهور تدريجي لصورة القسم
  - النص: يتحرك للأسفل قليلاً (translateY +4px)
  - يظهر زر "استكشف القسم ←" بـ fade-in
  - border-bottom: خط ملوّن بلون القسم
  - المدة: 300ms ease-in-out transition
```

### البيانات الديناميكية في الكارت (من API)

```typescript
// GET /api/departments  →  يُجلب في الصفحة الرئيسية بـ SSG
interface DepartmentCard {
  id: string;
  slug: string;
  name_ar: string;
  name_en: string;
  description_ar: string;    // جملة واحدة مختصرة
  faculty_count: number;     // عدد أعضاء التدريس
  cover_image_url: string;   // صورة تمثل القسم
  color: string;             // اللون المميز (يُدار من CMS)
  href: string;              // /departments/:slug
}
```

### الترابط مع باقي النظام
- الضغط على الكارت → صفحة القسم `/departments/:slug`
- صفحة القسم تجلب تلقائياً: أعضاء التدريس + البرامج + الأبحاث المرتبطة بهذا القسم
- `faculty_count` يتحدث تلقائياً عند إضافة/حذف عضو من القسم في الـ Admin Panel

## 2. المستخدمون والصلاحيات

### 2.1 أنواع المستخدمين

| النوع | الوصف | الصلاحيات |
|-------|--------|-----------|
| **زائر عام** | أي شخص يفتح الموقع بدون تسجيل | قراءة المحتوى العام فقط |
| **طالب** | طالب مسجل في الكلية | بوابة الطالب + الجداول + النتائج |
| **عضو هيئة تدريس** | محاضر / أستاذ | بوابة المحاضر + رفع المواد |
| **مدير محتوى** | موظف يدير محتوى قسم معين | تعديل محتوى قسمه فقط |
| **مدير النظام** | Super Admin | صلاحيات كاملة على كل الموقع |

### 2.2 آلية المصادقة (Authentication)

- تسجيل الدخول عبر **رقم الجامعة + كلمة مرور** للطلاب والمحاضرين
- تسجيل الدخول عبر **اسم المستخدم + كلمة مرور** للإداريين
- **JWT Tokens** للجلسات مع انتهاء صلاحية تلقائي (8 ساعات للإداريين، 24 ساعة للمستخدمين)
- **Two-Factor Authentication (2FA)** اختياري للطلاب، إلزامي للإداريين
- إعادة تعيين كلمة المرور عبر البريد الإلكتروني الجامعي فقط

## 4. تفصيل كل وحدة وظيفية

---

### M1 — وحدة محتوى الكلية

**الوصف:** تعرض المعلومات المؤسسية والإدارية للكلية.

#### M1.1 — الصفحة الرئيسية
- **الإدخال:** لا يوجد
- **المعالجة:** جلب آخر 5 أخبار + آخر 3 إعلانات + الفعاليات القادمة من قاعدة البيانات
- **الإخراج:** صفحة رئيسية ديناميكية تتحدث تلقائياً عند إضافة محتوى جديد
- **الترابط:** يقرأ من M1.2 (أخبار) + M6 (مؤتمرات) + M2 (إعلانات أكاديمية)

#### M1.2 — نظام الأخبار والإعلانات
- **الإدخال:** مدير المحتوى يضيف خبر (عنوان + نص + صورة + تاريخ النشر + تصنيف)
- **المعالجة:** حفظ في قاعدة البيانات + إرسال إشعار اختياري للمشتركين
- **الإخراج:** عرض الخبر في الصفحة الرئيسية + صفحة الأخبار + RSS Feed
- **الترابط:** يغذي M1.1 (الرئيسية) + M7 (محرك البحث)

#### M1.3 — معلومات الكلية
| الصفحة | المحتوى | من يعدلها |
|--------|--------|----------|
| تاريخ الكلية | نص + صور تاريخية | مدير النظام |
| رسالة الكلية | نص الرسالة | مدير النظام |
| رؤية الكلية | نص الرؤية | مدير النظام |
| قيادات الكلية | اسم + صورة + سيرة | مدير النظام |
| العمداء السابقون | قائمة تاريخية | مدير النظام |
| كلمة العميد | نص + صورة | مدير النظام |

- **الترابط:** بيانات قيادات الكلية مرتبطة بـ M4 (هيئة التدريس) — أي عضو تدريس بدرجة "عميد/وكيل" يظهر تلقائياً في صفحة القيادات

#### M1.4 — الأقسام العلمية الأربعة
كل قسم له صفحة تعرض:
- نبذة عن القسم
- رئيس القسم (مرتبط تلقائياً من M4)
- أعضاء التدريس في القسم (مرتبطون تلقائياً من M4)
- البرامج الدراسية في القسم (مرتبطة تلقائياً من M2)
- الأبحاث والمنشورات (مرتبطة تلقائياً من M4.3)

الأقسام: آثار مصرية — آثار إسلامية — ترميم الآثار — آثار يونانية رومانية

#### M1.5 — وحدات الكلية الخاصة
- **مركز صيانة الآثار:** صفحة مستقلة مع رابط للموقع الخارجي (cca-arch.cu.edu.eg)
- **متحف الكلية:** صفحة مستقلة مع رابط للموقع الخارجي
- **حفائر سقارة:** صفحة مستقلة مع رابط للموقع الخارجي

#### M1.6 — الاتصال بالكلية
- **الإدخال:** اسم + بريد إلكتروني + موضوع + رسالة + CAPTCHA
- **المعالجة:** التحقق من CAPTCHA + حفظ الرسالة في قاعدة البيانات + إرسال إشعار للإدارة
- **الإخراج:** رسالة تأكيد للمرسل + إشعار للجهة المختارة
- **الترابط:** قائمة الجهات المتاحة تُجلب من M4 (هيئة التدريس الإدارية)
- بيانات الاتصال الثابتة (فاكس + عنوان) تُدار من CMS

### M2 — وحدة الشأن الأكاديمي

**الوصف:** إدارة كل ما يتعلق بالبرامج الدراسية على مستوى البكالوريوس والدراسات العليا.

#### M2.1 — برامج البكالوريوس
- **الإدخال:** مدير المحتوى يضيف/يعدل البرامج الدراسية
- **البيانات المخزنة:** اسم البرنامج + القسم المرتبط + عدد الساعات + المقررات
- **الإخراج:** صفحة عرض لكل برنامج
- **الترابط:** كل برنامج مرتبط بقسم من M1.4 — تغيير القسم يحدث الروابط تلقائياً

#### M2.2 — المقررات الدراسية
- **البيانات المخزنة:** كود المقرر + الاسم + الوصف + عدد الساعات + الفصل + المتطلب السابق + القسم
- **الترابط:** كل مقرر مرتبط ببرنامج (M2.1) + أستاذ من M4 + جدول دراسي من M2.4

#### M2.3 — الجدول الدراسي
- **الإدخال:** مدير المحتوى يرفع الجدول (يدوياً أو عبر Excel Import)
- **المعالجة:** تخزين الجدول مع ربطه بالمقررات والأساتذة والقاعات
- **الإخراج:** عرض الجدول مصفى حسب القسم / الفرقة / اليوم
- **الترابط:** يقرأ من M2.2 (المقررات) + M4 (الأساتذة) — أي تغيير في بيانات الأستاذ ينعكس تلقائياً على الجدول

#### M2.4 — جدول الامتحانات ونتائجها

**جدول الامتحانات:**
- **الإدخال:** مدير المحتوى يرفع الجدول
- **الإخراج:** عرض الجدول للطلاب مصفى حسب الفرقة / القسم
- **الترابط:** مرتبط بالمقررات من M2.2 + بوابة الطالب في M3

**نتائج الامتحانات:**
- **الإدخال:** ملف Excel من شؤون الطلاب → يرفعه مدير النظام
- **المعالجة:** استيراد البيانات + ربط كل نتيجة برقم الطالب
- **الإخراج:** الطالب يدخل بوابته (M3) ويرى نتيجته فقط — لا يرى نتائج غيره
- **الترابط:** ربط مباشر مع M3 (بوابة الطالب) — النتيجة لا تظهر إلا بعد تسجيل الدخول

#### M2.5 — شروط القبول والالتحاق
- محتوى ثابت يعدله مدير النظام
- يعرض شروط القبول + المستندات المطلوبة + مواعيد التقديم
- **الترابط:** مرتبط بـ M2.1 (البرامج) — كل برنامج له شروط قبول مختلفة

#### M2.6 — برامج الدراسات العليا
نفس بنية M2.1 مع إضافات:
- متطلبات التسجيل في الماجستير والدكتوراه
- نماذج طلب الالتحاق (PDF قابل للتحميل)
- قائمة الرسائل العلمية المنجزة (مرتبطة من M4.3)

#### M2.7 — قواعد الساعات المعتمدة
- محتوى ثابت يعدله مدير النظام
- يعرض نظام الساعات المعتمدة + طريقة الاحتساب

### M3 — بوابة الطالب

**الوصف:** منطقة محمية بتسجيل الدخول تخص الطالب المسجل في الكلية.

#### M3.1 — تسجيل الدخول
- **الإدخال:** رقم الطالب الجامعي + كلمة المرور
- **المعالجة:** التحقق من قاعدة بيانات الطلاب + إصدار JWT Token
- **الإخراج:** توجيه الطالب لـ Dashboard الخاص به
- **الأمان:** تشفير كلمات المرور بـ bcrypt + Rate Limiting (5 محاولات فاشلة → حجب مؤقت 15 دقيقة)

#### M3.2 — Dashboard الطالب
يعرض بشكل مركزي:
- **الجدول الدراسي الحالي** (يُجلب تلقائياً من M2.3 بناءً على الفرقة والقسم)
- **جدول الامتحانات** (يُجلب تلقائياً من M2.4)
- **النتائج** (تُجلب من M2.4 مصفاة برقم الطالب)
- **الإعلانات المهمة** (تُجلب من M1.2 مصفاة بالأخبار الطلابية)
- **روابط سريعة** للخدمات المتاحة

#### M3.3 — خدمات الطالب
| الخدمة | الوصف | الترابط |
|--------|--------|---------|
| الجدول الدراسي | عرض الجدول الشخصي | M2.3 |
| نتائج الامتحانات | عرض النتائج الشخصية | M2.4 |
| جدول الامتحانات | عرض جدول القسم/الفرقة | M2.4 |
| مواد المقررات | تحميل ملفات PDF/PPT المرفوعة من المحاضر | M4.2 |
| الملف الشخصي | عرض/تعديل بيانات الاتصال | قاعدة بيانات الطلاب |
| تغيير كلمة المرور | تغيير آمن للكلمة السرية | نظام المصادقة |

### M3.4 — معالجة الملفات في الداشبورد (File Processing)

**المبدأ:** كل معالجة تحدث على الـ Backend (Node.js على السيرفر) — المتصفح فقط يرفع ويستقبل النتيجة.

#### Excel Import — الحالات المدعومة

| الملف | من يرفعه | المعالجة |
|-------|---------|---------|
| نتائج الطلاب (.xlsx) | مدير النظام | قراءة + Validation Report + حفظ بعد موافقة |
| الجدول الدراسي (.xlsx) | مدير المحتوى | قراءة + Validation Report + حفظ بعد موافقة |
| بيانات الطلاب (.xlsx) | مدير النظام | قراءة + Validation Report + حفظ بعد موافقة |
| فهرس المكتبة (.xlsx) | مدير المحتوى | قراءة + Validation Report + حفظ بعد موافقة |

**آلية الـ Validation Report:**
1. الملف يُرفع → Node.js يقرأه
2. يرجع تقرير: عدد الصفوف الصحيحة + قائمة الأخطاء بأرقام الصفوف
3. المدير يراجع → يوافق → الحفظ
4. لا حفظ إلا بعد موافقة صريحة

#### PDF — النوعان المدعومان

**النوع الأول — تخزين وتحميل (Storage & Serve):**
- المحاضر يرفع مذكرة / ورقة بحثية / نموذج تسجيل
- الملف يُخزَّن على السيرفر — لا استخراج للمحتوى
- الطالب/الزائر يضغط "تحميل" → يحصل على الملف مباشرة
- مناسب لـ: مواد المقررات + الأبحاث + ملفات المؤتمرات

**النوع الثاني — استخراج بيانات (Data Extraction):**
- المدير يرفع جدول امتحانات أو نتائج بصيغة PDF
- Node.js يستخرج النص من الـ PDF (**يعمل فقط مع PDF نصي — لا يعمل مع صور ممسوحة**)
- يُعرض للمدير البيانات المستخرجة للمراجعة قبل الحفظ
- إذا فشل الاستخراج: يظهر تحذير ويُتاح للمدير الإدخال اليدوي أو رفع Excel بديلاً

### M4 — وحدة هيئة التدريس

**الوصف:** إدارة بيانات أعضاء هيئة التدريس وربطها بكل أجزاء الموقع.

#### M4.1 — قاعدة بيانات أعضاء هيئة التدريس
**البيانات المخزنة لكل عضو:**
- الاسم (عربي + إنجليزي)
- الدرجة العلمية (مدرس مساعد / مدرس / أستاذ مساعد / أستاذ)
- القسم المنتسب إليه
- التخصص الدقيق
- البريد الإلكتروني الجامعي
- صورة شخصية
- السيرة الذاتية المختصرة
- المنصب الإداري إن وجد (عميد / وكيل / رئيس قسم)

**الترابط الأوتوماتيكي:**
- أعضاء التدريس في كل قسم → تظهر تلقائياً في صفحة القسم (M1.4)
- من لديه منصب إداري → يظهر تلقائياً في صفحة القيادات (M1.3)
- الأستاذ المرتبط بمقرر → يظهر تلقائياً في بيانات المقرر (M2.2)

#### M4.2 — بوابة المحاضر (Login)
- **تسجيل دخول:** بريد إلكتروني جامعي + كلمة مرور
- **الصفحة الرئيسية:** عرض المقررات المرتبطة به
- **الوظائف:**
  - رفع المواد الدراسية (PDF/PPT/DOCX) مرتبطة بمقرر محدد
  - **نوع الرفع:** تخزين وتحميل فقط — الملف يُخزَّن على السيرفر ويستطيع الطالب تحميله
  - رفع الإعلانات الخاصة بمقرراته
  - تعديل بياناته الشخصية وسيرته الذاتية
- **الترابط:** المواد المرفوعة تظهر تلقائياً لطلاب المقرر في M3
- **حدود الرفع:** PDF حتى 25MB | PPT حتى 50MB | صور حتى 10MB

#### M4.3 — الأبحاث والمنشورات
- **الإدخال:** عضو التدريس يرفع بيانات بحثه (عنوان + ملخص + سنة النشر + رابط/PDF)
- **الإخراج:** تظهر في صفحة الباحث الشخصية + صفحة قسمه (M1.4) + المجلة العلمية (M4.4)
- **الترابط:** يُغذي M7 (محرك البحث) تلقائياً

#### M4.4 — المجلة العلمية للكلية
- قائمة بأعداد المجلة (كل عدد = مجموعة أبحاث منشورة)
- كل بحث مرتبط بصاحبه من M4.1
- إمكانية تحميل الأبحاث كـ PDF
- **الترابط:** يُغذي M7 (محرك البحث) تلقائياً

### M5 — وحدة المكتبة

**الوصف:** واجهة إلكترونية لمكتبات الكلية الأربع.

#### M5.1 — الفهرس الإلكتروني
- كل مكتبة لها فهرس منفصل في قاعدة البيانات
- **البيانات:** عنوان الكتاب + المؤلف + سنة النشر + الناشر + عدد النسخ + القسم
- **الترابط:** الفهرس مرتبط بـ M7 (محرك البحث) — الزائر يبحث عن كتاب من أي صفحة

المكتبات:
1. مكتبة الآثار المصرية
2. مكتبة الآثار الإسلامية
3. مكتبة ترميم الآثار
4. مكتبة الدراسات العليا والبحوث

#### M5.2 — روابط المكتبات الخارجية
قائمة روابط تُدار من CMS:
- مكتبات الجامعات المصرية (EUL)
- IEEE Xplore
- ScienceDirect
- إضافة روابط جديدة بدون تعديل الكود

### M6 — وحدة المؤتمرات

**الوصف:** إدارة المؤتمرات الدولية للكلية مع إمكانية إضافة مؤتمرات جديدة مستقبلاً.

#### M6.1 — هيكل المؤتمر
كل مؤتمر هو **entity مستقلة** تحتوي على:
- اسم المؤتمر (عربي + إنجليزي)
- رقم المؤتمر (الخامس، السادس...)
- تاريخ الانعقاد
- الموضوع الرئيسي
- شعار المؤتمر (Banner Image)
- حالة المؤتمر (قادم / جارٍ / منتهٍ)

#### M6.2 — صفحات المؤتمر
كل مؤتمر يولد تلقائياً هذه الصفحات:
| الصفحة | المحتوى | من يديرها |
|--------|--------|----------|
| الرئيسية | مقدمة + أهداف المؤتمر | مدير المحتوى |
| التسجيل | نموذج تسجيل المشاركين | مدير المحتوى |
| الملخصات | رفع واستعراض الملخصات | مدير المحتوى + المشاركون |
| المحاور العلمية | محاور البحث المقبولة | مدير المحتوى |
| متطلبات الأبحاث | تعليمات للباحثين | مدير المحتوى |
| المواعيد | جدول زمني للمؤتمر | مدير المحتوى |
| رسوم الاشتراك | جدول الرسوم | مدير المحتوى |
| البرنامج | جدول الجلسات | مدير المحتوى |
| الأبحاث المقدمة | قائمة الأبحاث المقبولة | مدير المحتوى |

#### M6.3 — نموذج التسجيل في المؤتمر
- **الإدخال:** اسم + مؤسسة + بريد إلكتروني + عنوان البحث + ملخص البحث + نوع المشاركة
- **المعالجة:** حفظ البيانات + إرسال بريد تأكيد تلقائي للمشارك + إشعار للإدارة
- **الإخراج:** رقم تسجيل فريد للمشارك
- **الترابط:** البيانات تدخل قاعدة بيانات المؤتمر المرتبطة بـ M6.1

#### M6.4 — أرشيف المؤتمرات
- جميع المؤتمرات السابقة تُحفظ وتبقى متاحة
- المؤتمر الخامس (2017) والسادس (2018) يُنقلان للأرشيف
- **الترابط:** الأرشيف يُغذي M7 (محرك البحث)

#### M6.5 — إضافة مؤتمر جديد
- مدير النظام يضيف مؤتمر جديد من لوحة التحكم
- الصفحات تُولَّد تلقائياً بدون تدخل تقني
- **الترابط:** المؤتمر الجديد يظهر تلقائياً في قائمة المؤتمرات + الصفحة الرئيسية

### M7 — محرك البحث المركزي

**الوصف:** نظام بحث موحد يغطي كل محتوى الموقع.

#### M7.1 — النطاق
محرك البحث يفهرس تلقائياً:
- أخبار الكلية (من M1.2)
- أعضاء هيئة التدريس (من M4.1)
- الأبحاث والمنشورات (من M4.3)
- المقررات الدراسية (من M2.2)
- الكتب في المكتبة (من M5.1)
- صفحات المؤتمرات (من M6)

#### M7.2 — آلية البحث
- **الإدخال:** نص البحث من أي صفحة في الموقع
- **المعالجة:** البحث في كل المحتوى المفهرس مع دعم اللغة العربية (stemming)
- **الإخراج:** نتائج مصنفة حسب النوع (خبر / شخص / بحث / كتاب / مقرر)
- **الفلترة:** تصفية النتائج حسب النوع / التاريخ / القسم

## 6. متطلبات اللغة والـ i18n

- الموقع يعمل بـ **لغتين: العربية والإنجليزية**
- الاتجاه: عربي → RTL، إنجليزي → LTR
- **كل محتوى في قاعدة البيانات له حقلان:** `content_ar` و `content_en`
- تبديل اللغة يغير المحتوى + الاتجاه + التاريخ (هجري/ميلادي) في نفس الوقت
- إذا لم يتوفر ترجمة إنجليزية → يعرض المحتوى العربي كـ fallback مع علامة تحذيرية في لوحة التحكم

## 7. متطلبات الأمان الوظيفي

| المتطلب | التفاصيل |
|--------|---------|
| **HTTPS إلزامي** | كل الطلبات تُحول تلقائياً من HTTP إلى HTTPS |
| **Rate Limiting** | 100 طلب / دقيقة للزوار، 10 محاولات تسجيل دخول / 15 دقيقة |
| **Input Validation** | كل المدخلات تُعقم على مستوى الـ Backend قبل حفظها |
| **Parameterized Queries** | لا SQL مباشر — كل استعلامات قاعدة البيانات عبر ORM |
| **File Upload Security** | ملفات الرفع: PDF/PPT/JPG فقط، حجم أقصى 10MB، فحص MIME type |
| **Audit Log** | كل عملية كتابة في الـ CMS تُسجل (من + متى + ماذا فعل) |
| **Session Invalidation** | عند تغيير كلمة المرور → إلغاء كل الجلسات النشطة الأخرى |
| **PII Protection** | أرقام هواتف الإداريين لا تظهر كنص عادي — تُعرض كـ click-to-call |

## 10. السيناريوهات الوظيفية الكاملة (Use Cases)

### UC-01: طالب يريد معرفة نتيجة امتحانه
1. الطالب يفتح الموقع → يضغط "بوابة الطالب"
2. يدخل رقمه الجامعي + كلمة المرور
3. النظام يتحقق من هويته → يفتح Dashboard
4. Dashboard يعرض تلقائياً نتائجه المرتبطة برقمه
5. الطالب يضغط "النتائج التفصيلية" → يرى جميع نتائجه

### UC-02: مدير المحتوى يضيف خبراً جديداً
1. مدير المحتوى يدخل لوحة التحكم
2. يختار "أخبار وإعلانات" → "إضافة خبر جديد"
3. يملأ النموذج (عنوان عربي + إنجليزي + نص + صورة + تصنيف)
4. يضغط "نشر"
5. الخبر يظهر تلقائياً في: الصفحة الرئيسية + صفحة الأخبار + نتائج البحث

### UC-03: مدير النظام يضيف عضو تدريس جديد
1. مدير النظام يدخل "هيئة التدريس" → "إضافة عضو"
2. يملأ البيانات (اسم + درجة + قسم + منصب إداري إن وجد)
3. يضغط "حفظ"
4. العضو يظهر تلقائياً في:
   - صفحة قسمه (M1.4)
   - صفحة القيادات إذا كان له منصب (M1.3)
   - قائمة المقررات المتاحة للربط (M2.2)

### UC-04: إضافة مؤتمر جديد
1. مدير النظام يدخل "المؤتمرات" → "مؤتمر جديد"
2. يملأ بيانات المؤتمر الأساسية
3. يضغط "إنشاء"
4. النظام يولد تلقائياً جميع صفحات المؤتمر (8 صفحات)
5. مدير المحتوى يملأ تفاصيل كل صفحة
6. المؤتمر يظهر في الصفحة الرئيسية تلقائياً

### UC-05: باحث خارجي يبحث عن ورقة بحثية
1. الباحث يكتب في خانة البحث
2. محرك البحث يجلب النتائج من: أبحاث المجلة + ملخصات المؤتمرات + فهرس المكتبة
3. الباحث يصفي النتائج بـ "أبحاث فقط"
4. يضغط على نتيجة → ينتقل لصفحة الباحث أو تحميل الملف
